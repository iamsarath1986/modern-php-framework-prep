Two modules Users and Orders. These modules should be designed to handle large-scale data, potentially millions of records. Therefore, 
the system should include efficient search functionality (similar to Elasticsearch), a queue system for background processing, and proper performance optimization.

Each module will include APIs to list, create, update, and delete records. The APIs must support pagination, advanced search capabilities, 
and caching of search results to improve performance.
